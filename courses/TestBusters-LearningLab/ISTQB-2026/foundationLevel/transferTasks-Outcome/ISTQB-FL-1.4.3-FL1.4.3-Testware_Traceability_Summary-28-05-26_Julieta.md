# Definitions

## Testware

Testware includes all work products created during testing, such as:

* test concepts
* test plans
* test data
* scripts
* reports

---

## Traceability

Traceability connects:

* requirements
* test cases
* risks
* results

It supports:

* coverage analysis
* audits
* change impact analysis

# Test Activities & Testware


| Test Activity                 | Work Products / Testware                                                                         |
| ----------------------------- | ------------------------------------------------------------------------------------------------ |
| **Test planning**             | test schedule, test plan, risk register, entry & exit criteria                                   |
| **Test monitoring & control** | test progress reports, documentation of control directives, information about risks              |
| **Test analysis**             | test conditions, defect reports                                                                  |
| **Test design**               | test cases, test charters, coverage items, test data requirements, test environment requirements |
| **Test implementation**       | test procedures, manual and automated test scripts, test suites, test data, execution schedule   |
| **Test execution**            | test logs, defect reports                                                                        |
| **Test completion**           | test completion report, documented lessons learned, change requests                              |


# Examples Testware

| Test activity       | Testware               |
| ------------------- | ---------------------- |
| Test planning       | Test schedule          |
| Test analysis       | Test conditions        |

| Test execution      | Test logs              |

Each test activity produces specific work products.
These testware help teams to carry out tests in a structured, traceable and repeatable way.

# Traceability

Traceability means linking:

* requirements
* test cases
* test results
* risks

This enables the team to understand:

* which requirements have been tested
* which test cases are linked to specific requirements
* what test results were produced
* which risks or gaps still exist

## Benefits of Traceability

* improves coverage analysis
* supports change impact analysis
* increases transparency
* supports audits and reporting
* improves communication within Scrum teams

# Traceability Example

| Requirement                                           | Test case                            | Result                           | Traceability purpose                        |
| ----------------------------------------------------- | ------------------------------------ | -------------------------------- | ------------------------------------------- |
| Search should display matching products               | Search for “Screwdriver”             | Matching products are displayed  | Verify requirement coverage                 |
| Users should be able to log in with valid credentials | Login with valid email and password  | User is successfully logged in   | Ensure authentication requirement is tested |
| Cart should store selected products                   | Add product to cart                  | Product appears in cart          | Validate shopping cart functionality        |
| System should reject invalid login attempts           | Login with wrong password            | Error message is displayed       | Verify negative scenario coverage           |
| Product details should be visible                     | Open product detail page             | Product information is displayed | Ensure UI and requirement validation        |
| Checkout requires customer information                | Submit checkout form with valid data | Order is successfully submitted  | Validate end-to-end business process        |


# Conclusion

Effective testware supports:

* consistent testing
* repeatable test execution
* well-documented testing processes

This helps teams:

* improve software quality
* reduce project risks
* increase testing transparency

---

## Traceability Chain

Traceability works like a connection chain:

**Requirement → Test → Result**

This connection helps teams understand:

* which requirements were tested
* which test cases belong to specific requirements
* what results were produced
* which risks or gaps still exist

---

## Benefits of Good Traceability

* better transparency
* improved coverage assessment
* easier change impact analysis
* better team collaboration
* clearer reporting and audits
